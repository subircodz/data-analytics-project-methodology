# DAPM PROJECT CHECKLIST

Use this checklist to track progress throughout the project lifecycle.

---

# Phase 1 — Business Understanding

- [ ] Understand the business domain.
- [ ] Define the business problem.
- [ ] Identify business objectives.
- [ ] Understand current business processes.
- [ ] Document assumptions.
- [ ] Record project scope.

---

# Phase 2 — Stakeholder Analysis

- [ ] Identify stakeholders.
- [ ] Understand stakeholder responsibilities.
- [ ] Identify business decisions.
- [ ] Document business questions.
- [ ] Define required information.

---

# Phase 3 — Business Requirements Gathering

- [ ] Identify metrics.
- [ ] Distinguish KPIs from metrics.
- [ ] Define project success criteria.
- [ ] Document analytical requirements.

---

# Phase 4 — Data Discovery & Collection

- [ ] Identify data sources.
- [ ] Obtain required datasets.
- [ ] Verify data accessibility.
- [ ] Document dataset inventory.

---

# Phase 5 — Data Validation

- [ ] Validate missing values.
- [ ] Check duplicate records.
- [ ] Verify data types.
- [ ] Identify inconsistent values.
- [ ] Review outliers.
- [ ] Complete Validation Report.

---

# Phase 6 — Data Cleaning & Preparation

- [ ] Handle missing values.
- [ ] Remove duplicates.
- [ ] Standardize formats.
- [ ] Merge datasets.
- [ ] Prepare analysis-ready dataset.

---

# Phase 7 — Exploratory Data Analysis (EDA)

- [ ] Perform descriptive statistics.
- [ ] Explore distributions.
- [ ] Analyze relationships.
- [ ] Create supporting visualizations.
- [ ] Document observations.
- [ ] Raise business questions.

---

# Phase 8 — Business Insights

- [ ] Interpret observations.
- [ ] Connect findings to business context.
- [ ] Identify risks.
- [ ] Identify opportunities.
- [ ] Produce Business Insights Report.

---

# Phase 9 — Recommendations

- [ ] Develop recommendations.
- [ ] Assign priorities.
- [ ] Define implementation owners.
- [ ] Estimate expected benefits.
- [ ] Complete Recommendations Report.

---

# Phase 10 — Reporting

- [ ] Prepare Executive Summary.
- [ ] Review documentation.
- [ ] Finalize README.
- [ ] Update Project Journal.
- [ ] Complete portfolio artifacts.

---

# Project Closure

- [ ] Verify all phases completed.
- [ ] Review methodology compliance.
- [ ] Record lessons learned.
- [ ] Identify DAPM improvements.
- [ ] Archive project.
- [ ] Update DAPM if this project revealed improvements.

---
